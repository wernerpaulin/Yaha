Add-ons

Yaha is fully modular and can be extended by various add-ons. Each add-on comes with its own user interface.
Additionally modules can be distributed on several hardware devices to improve availability.
The following modules exist already:

Overview - dashboard

This is actually no runtime module but a dashboard module in the UI. It provides quick access to frequently used controls such as all shutters up or down, current temperatures, etc. The dashboard also features diagnostics page and system settings page for language switching and date & time settings.
Screenshots

Blinds and shading

With this module any kind of shading device can be automated. Currently rolling shutters are supported. To control the angle of blinds is currently not implemented. Several devices can be grouped freely and can be operated together (e.g. all ground floor shutters up or down).
Screenshots

Features:
  • Open / close / stop
  • Move to manually given set position
  • Auto shading using sun position
  • Flexible time control to open or close like entire year a period, weekend or day of week
  • Event control like open and sun rise

Weather

This modules gathers any kind of data from a various given weather stations which has a web interface and is accessible through WiFi or Ethernet. In other words if you can browse life data on your weather station this module can do it too: It mimics a HTML page request to the weather station and interprets the HTML response. The renkforce WH2600 weather station is a cost-efficient entry version from conrad.at and has been successfully tested.
Screenshots

Features:
  • Time stamp
  • Indoor temperature
  • Outdoor temperature and trend
  • Outdoor temperature "feels like" with wind chill and heat index
  • Indoor humidity
  • Outdoor humidity and trend
  • Absolute pressure
  • Relative pressure and trend
  • Wind direction
  • Wind speed
  • Wind gust
  • Solar radiation
  • UV radiation
  • UV radiation
  • Weather condition summary (sunny, cloudy, rain)
  • Together with the recorder module all these values can be recorded with time stamp

Recorder

This module is a runtime module. It records continously and synchronous to the cyclic system any given data points and creates a CSV file. Several recorders can be scheduled in parallel.
The user interface uses D3 (Data-Driven Documents) to display the data.
Screenshots

Features:
  • Minimum recorder interval: 1s
  • Trigger to start recording by time or command
  • Number formating before saving to CSV
  • Average calculation of data point between scans

Lights, doors, gates and switches

This module is very versatile and can control any on/off switch for lights or doors. Although not yet fully implemented it is also prepared to control hue lights. On and off commands can be initiated by different signals. Additionally each command can be a logical (and/or) combination of several signals.
Screenshots

Features:
  • Inputs and outputs can be normally open or normally closed
  • Blink mode with independent on and off durations
  • Actuator can be configured as impulse with a certain on-duration
  • Actuator can have an off-delay which delays the switch off for a certain time once the off-command has been initiated (e.g. for lights).
  • Flexible time control to switch on and off at a certain time in a defined period, weekend or day of week
  • Energy measurement (if actuator provides these data)

Heating

The heating module allows you to control both the heating system and hot water preparation. The heating system consits of a number of heating circuits in your home. Each circuit is independently controlled individually by a weather-compensated closed-loop control: a certain outside temperature and a desired indoor temperature are used to calculate a certain set flow temperature. A PI-controller ensures that the actual flow temperature matches the set flow temperature.
Screenshots

Each heating circuit has:
  • A heat source (the form of energy is not important. Can be oil, gas, heat pump, solar,...)
  • A pump which brings the warm media (e.g. water) into the building
  • A mixer which is PI controlled so the flow temperature keeps a certain set point
Features of a heating circuit:
  • Modes of each circuit: automatic (time controlled), permanent comfort operation, permanent reduced operation
  • Temperature settings for comfort and reduced operation
  • All circuits can be switched globally on, off, suspended ventilation, not at home
  • Different timings for weekdays and weekends
  • Heat curve to calculate flow temperature can be manipulated by level and slope
  • PI controller for mixer
  • 2-point hysteresis controller for room temperature
  • Manual operation to get full control of the mixer
  • Simulation of entire circuit
Features of hot water preparation:
  • Free pump assignment
  • Time controlled in which a set temperature is controlled
  • Legionella protection
  • Simulation of entire module
Features of heat source:
  • Heating device (e.g. burner) is controlled by a digital release signal
  • Digital status feedback for general failure and overtemperature
  • One heat source can be a supply for various heat circuits
  • Safety switch off
  • Manual operation
  • Simulation of heat source behaviour
Features of mixers:
  • Open and close by different outputs
  • Actual position estimation
  • Automatic homing every midnight
  • Manual operation in combination with heat circuit
  • Simulation of mixer movement
Features of pumps:
  • Frost protection
  • Safety release monitoring (external temperature sensors which can override pump)
  • Manual operation
  • Simulation of pump behaviour
